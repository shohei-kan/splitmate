# Project Context

## Project name
SplitMate

## Purpose
A household expense-sharing app for couples/families.

## Stack
- Backend: Django, Django REST Framework
- Frontend: React, Vite
- DB: Postgres
- Dev environment: Docker Compose

## Directory overview
- `backend/`: API and backend logic
- `frontend/`: UI and client-side logic
- `docs/`: project notes and Codex/GPT handoff files

## Working style
- Keep changes small and reviewable
- Prefer understanding the existing implementation before editing
- Match current conventions
- Minimize regressions
- Use Japanese for end-user UI where appropriate

## Current priorities
- Improve UI clarity
- Strengthen backend reliability
- Keep frontend/backend contracts stable
